# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Is

A Claude Code session reporting tool. It scans all Claude Code sessions across every project on the machine, generates AI-powered summaries, and renders an HTML dashboard showing what was worked on, status, and next steps.

The tool is designed as a **Claude Code slash command** (`/session-report`) that orchestrates a multi-step pipeline: collect sessions ‚Üí summarize via Claude ‚Üí cache ‚Üí render HTML report.

## Architecture

**Single-file Python script** (`claude-session-report.py`) with no external dependencies beyond the standard library. It handles:

- **Session scanning**: Reads JSONL transcript files from `~/.claude/projects/` and cross-references `~/.claude/sessions/` and `~/.claude/history.jsonl` for metadata
- **Caching**: Summary cache at `~/.claude/session-report-cache.json`, keyed by session ID with message count + timestamp for invalidation
- **Transcript sampling**: For long sessions (>20 messages), samples first 6, middle 8, and last 6 messages to stay within token limits (12k char cap)
- **HTML rendering**: Self-contained HTML report with inline CSS/JS, written to `~/.claude/reports/` and opened in browser

**Slash command definition** (`session-report.md`) orchestrates the pipeline by instructing Claude Code to:
1. `--collect` ‚Üí outputs `_collect.json` with sessions needing summaries
2. Claude reads transcripts and writes summaries to `_summaries.json`
3. `--update-cache` merges summaries into the persistent cache
4. `--review-statuses` ‚Üí enables cross-session status correction (e.g., marking earlier sessions as `handed_off`)
5. Final render pass generates the HTML report

**Setup instructions** (`session-report-setup.md`) is a one-time install guide meant to be given to Claude Code.

## Running

```bash
# Generate HTML report (last 3 days)
python claude-session-report.py

# Pipeline subcommands (used by the slash command)
python claude-session-report.py 3 --collect
python claude-session-report.py --update-cache ~/.claude/reports/_summaries.json
python claude-session-report.py 3 --review-statuses
python claude-session-report.py --update-statuses ~/.claude/reports/_status_corrections.json

# Flags (bare words work without --)
python claude-session-report.py 7 text       # terminal output
python claude-session-report.py 3 no-ai      # skip AI summaries
python claude-session-report.py 3 refresh     # force re-summarize
python claude-session-report.py 14 --limit 20 # cap session count
python claude-session-report.py 1 --no-open   # don't open browser
```

## Key Design Decisions

- **Encoding on Windows:** Always use `encoding="utf-8"` on all `open()` calls. The system default is cp1252, which corrupts em dashes and other multi-byte UTF-8 characters. The slash command's merge step is the main risk area ‚Äî see `session-report.md` Step 3.
- **Summaries are generated by Claude Code, not the script.** The script only collects data and renders. The slash command prompt instructs Claude to read transcripts and produce structured summaries with status/title fields.
- **Session statuses** (`complete`, `in_progress`, `blocked`, `handed_off`) are assigned during summarization and refined in a cross-session review pass that looks at related sessions in the same folder.
- **Cache invalidation** uses `(msg_count, latest_ts)` tuples ‚Äî a session is re-summarized only if its message count or latest timestamp changes.
- **Windows paths** are handled throughout: `decode_project_path()` reconstructs drive letters from the `--`-encoded directory names Claude Code uses internally.
- **The setup doc references a LiteLLM proxy** (`ANTHROPIC_BASE_URL`) for AI summaries ‚Äî this is from an older version. The current `session-report.md` has Claude Code do the summarization directly.

## Reading the Script

`claude-session-report.py` is ~2200+ lines ‚Äî too large to read in one pass. Line numbers drift frequently; use `grep -n` to find sections rather than relying on hardcoded ranges. Key landmarks:
- `CACHE_FILE` and cache functions (near top)
- `aggregate_projects()` ‚Äî project-level status/metadata
- `generate_html()` ‚Äî main HTML output entry point
- `toggleInactiveVisibility` ‚Äî JS sidebar logic (search for `function` keyword)
- `argparse` setup (near bottom)

## Working with the HTML Template

- **Sidebar structure:** Unified recency-sorted project list (no filter chips). Active/inactive items mixed, inactive dimmed. Checkbox toggle in footer hides inactive. Project status badge dropdown allows manual Active/Inactive override (stored in localStorage key `sr-project-overrides`).
- **localStorage keys:** `sr-theme`, `sr-sidebar-collapsed`, `sr-corrections`, `sr-project-overrides`, `sr-hide-inactive`
- The HTML template is embedded in `claude-session-report.py` as an f-string ‚Äî all CSS `{` `}` must be doubled (`{{` `}}`)
- The template uses a **sidebar + detail panel (master-detail) layout**: the sidebar lists projects, and clicking a project renders its sessions in the detail panel
- **JS renders project detail client-side** from embedded JSON ‚Äî session data is serialized into a `<script type="application/json" id="projects-data">` block, and JS builds the detail view dynamically
- **Status corrections** from a previous report are embedded in a `<script type="application/json" id="status-corrections">` block and carried forward automatically
- Key server-side functions for the new layout:
  - `aggregate_projects()` (~812) ‚Äî groups sessions by project folder, computes project-level status/metadata, splits into active vs. inactive
  - `parse_summary_sections()` (~749) ‚Äî splits a summary into titled sections (What Was Done, Current Status, etc.) for structured rendering
  - `read_corrections_from_previous_report()` (~117) ‚Äî reads the most recent HTML report and extracts embedded status corrections JSON so they persist across regenerations
- `md_to_html()` (~692) converts summary markdown to HTML ‚Äî heading levels here control the document's heading hierarchy
- To preview locally, use `start "" <path>` on Windows ‚Äî the Chrome MCP prepends `https://` to `file:///` URLs
- For Playwright MCP preview: `python -m http.server 8787 --directory ~/.claude/reports` then navigate to `localhost:8787/<filename>`

## Deployment

After editing source files, copy to installed locations:
```bash
cp claude-session-report.py ~/.claude/scripts/claude-session-report.py
cp session-report.md ~/.claude/commands/session-report.md
```
Verify with `python claude-session-report.py 3 --no-open`, then preview with Playwright via local HTTP server.
